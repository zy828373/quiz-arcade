# Claude Code CLI 接入号池指南

> 适用场景：同一局域网内的电脑，连接 192.168.8.200 的 GPT Team 号池
> 服务器固定IP：192.168.8.200 | 号池端口：8317

---

## 前置条件检查

在开始之前，确认以下两点：

1. **和服务器在同一WiFi/局域网**
2. **浏览器能访问** `http://192.168.8.200:8317/v1/models`（能看到JSON就行）

---

## 第一步：安装依赖软件

claude-bridge 和 Claude Code 需要以下两个软件，**缺一不可**：

### 1. Git for Windows（必须）

Claude Code 在 Windows 上必须依赖 Git Bash，否则启动时报错。

- 下载地址：https://git-scm.com/downloads/win
- 无脑下一步安装，**安装完重启终端**
- 验证：`git --version` 能看到版本号

### 2. Node.js（必须）

claude-bridge 需要 Node.js 环境（建议 v18+）

- 下载地址：https://nodejs.org/en/download
- 安装完后重启终端，验证：`node -v` 和 `npm -v` 能看到版本号

---

## 第二步：安装官方 Claude Code CLI

> ⚠️ 注意：必须是官方版，不能是魔改版（魔改版启动时会显示"中转服务器: xxx.yhlxj.com"）

```powershell
# 如果装了魔改版，先卸载
npm uninstall -g claude-code

# 安装官方版
npm install -g @anthropic-ai/claude-code
```

验证安装：
```powershell
claude --version
# 应显示 Claude Code v2.x.xx（不含中转服务器提示）
```

---

## 第三步：配置 Claude Code 跳过登录

`.claude` 文件夹首次安装时不存在，直接在 PowerShell 执行以下命令**自动创建文件夹和配置文件**：

```powershell
New-Item -Path "$env:USERPROFILE\.claude" -ItemType Directory -Force
@'
{
  "env": {
    "ANTHROPIC_BASE_URL": "http://localhost:8000",
    "ANTHROPIC_AUTH_TOKEN": "dummy",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1"
  }
}
'@ | Out-File -FilePath "$env:USERPROFILE\.claude\settings.json" -Encoding utf8
```

执行完后可在 `C:\Users\<用户名>\.claude\settings.json` 看到这个文件。

---

## 第四步：每次使用前启动 claude-bridge

**必须先启动 claude-bridge，再开 Claude Code。**

打开 PowerShell，执行（保持这个窗口不关闭）：

```powershell
npx -y claude-bridge -u http://192.168.8.200:8317/v1 -m gpt-5.4 -k team-api-key-1
```

看到以下输出说明启动成功：
```
✅ Server running at: http://localhost:8000
🎯 Target API: http://192.168.8.200:8317/v1
🤖 Model: gpt-5.4
```

---

## 第五步：启动 Claude Code

重新打开一个 PowerShell 窗口，执行：

```powershell
claude
```

正常情况下会直接进入 Claude Code 界面，显示 `Sonnet 4.6`，无需登录。

---

## 故障排查

| 现象 | 原因 | 解决方法 |
|------|------|---------|
| `Claude Code on Windows requires git-bash` | 未安装 Git for Windows | 安装 Git（第一步1）后重启终端 |
| 弹出浏览器登录页，显示"中转服务器: xxx.yhlxj.com" | 装了魔改版 | 卸载重装官方版（第二步）|
| `There's an issue with the selected model` | claude-bridge未启动 | 先执行第四步再跑claude |
| claude-bridge启动失败，报连接错误 | 和服务器不在同一局域网 | 确认WiFi，或用公网地址 |
| `Missing API key`（访问8317时浏览器显示） | 正常，浏览器不带Authorization头 | 不是问题，继续下一步 |
| Auth conflict警告 | 系统有残留的ANTHROPIC_*环境变量 | `Remove-Item Env:ANTHROPIC_API_KEY -EA SilentlyContinue` |
| `EADDRINUSE: address already in use` | 旧进程未退出仍占用端口(8000/8320等) | 管理员CMD: `netstat -ano \| findstr :端口号` 找PID → `taskkill /F /PID <PID>` |
| `start_all.bat` 双击后全是乱码 | bat文件UTF-8编码与CMD的GBK冲突 | bat开头加 `chcp 65001 >nul 2>&1`，或中文改英文 |
| Tunnel报 `failed to dial to edge with quic: timeout` | Clash TUN模式劫持QUIC(UDP)流量 | `config.yml` 加 `protocol: http2` |

---

## 公网访问（不在同一局域网时）

如果不在公司WiFi，把第四步的地址换成 Cloudflare 公网地址：

```powershell
npx -y claude-bridge -u https://api.codexapis.uk/v1 -m gpt-5.4 -k team-api-key-1
```
